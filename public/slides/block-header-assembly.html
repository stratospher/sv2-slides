Byte Ordering

The merkle_root and other fields must be serialized in Little-Endian format (byte-reversed). This is a common pitfall in implementation.

The Mining Goal

SHA256(SHA256(Header)) < Target

Miners iterate the nonce. If it overflows, they update ntime or rebuild the coinbase (changing merkle_root via extranonce2).

SHA256 Padding

The 80-byte header is padded to 128 bytes (chunk size) for the hash function.

Search Space

The 32-bit nonce only provides ~4 billion values. At modern hashrates, this is exhausted in milliseconds, forcing frequent extranonce2 updates.

Security Note: In SV1, the pool constructs most of this header. The miner only iterates the nonce and a small part of the coinbase (extranonce2), blindly trusting the rest.
