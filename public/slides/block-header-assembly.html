Byte Ordering

The merkle_root and other fields must be serialized in Little-Endian format (byte-reversed). A common implementation pitfall.

Raw: a1b2... → Serialized: ...b2a1
The Mining Goal

SHA256(SHA256(Header)) < Target

Miners iterate the nonce. If it overflows, update ntime or rebuild the coinbase (changing merkle_root via extranonce2).

SHA256 Padding

The 80-byte header is padded to 128 bytes (chunk size) for the hash function.

Search Space

The 32-bit nonce provides ~4 billion values — exhausted in milliseconds at modern hash rates, forcing frequent extranonce2 updates.

Security Note: In SV1, the pool constructs most of this header. The miner only iterates the nonce and extranonce2 — blindly trusting the rest.
