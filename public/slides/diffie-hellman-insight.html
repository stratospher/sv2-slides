Key Exchange: The Diffie–Hellman Insight

1

Generate Private Keys

Each party generates a random private number (k). This is the "seed" for the entire security model.

Never shared across the network.

2

Derive & Exchange Public Keys

Derive public key P = k ∘ G using a one-way function (Elliptic Curve). These are safe to send openly.

Visible to MITM attackers.

3

Compute Shared Secret

Combine your private key with the peer's public key mathematically to arrive at the same secret result.

S = k1 ∘ P2 = k2 ∘ P1

The Magic: Even though the attacker sees both public keys P1 and P2, they cannot calculate S without knowing at least one private key (k1 or k2).
