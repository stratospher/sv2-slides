Server Authentication in SV2

1

Authority Key Distribution

Mining pools publish their static Authority Public Key via trusted out-of-band channels (website, DNS). Miners configure/pin this key before connecting.

2

Noise Handshake Verification

During the handshake (e.g., Noise_XK), the miner verifies the remote server's static key matches the pinned Authority Key.

3

Cryptographic Proof

The protocol requires the server to use the corresponding Private Key to derive session keys. Without the private key, the handshake mathematics fail immediately.

Security Impact

This effectively blocks MITM attacks. An attacker can intercept the TCP connection, but without the pool's private key, they cannot complete the handshake or impersonate the server.
