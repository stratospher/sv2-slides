PROTOCOL MESSAGE // STEP 02

mining.authorize

Authenticates the worker to the mining pool

Authentication

After subscribing, the miner must authorize a worker to submit shares. This binds the connection to a specific user account on the pool.

  • Worker Name: Usually format "username.worker". Allows tracking individual devices.
  • Password: Often optional or generic (e.g., "x") â€” security relies on the payout address, not the worker login.

Security Note

No Cryptographic Binding: This authentication does not integrity-protect the session. An attacker can hijack the TCP stream after authorization and submit valid shares for their own worker, or inject jobs.

Client Request 
// Miner authorizes specific worker
{
  "id": 2,
  "method": "mining.authorize",
  "params": [
    "user_account.worker1", // Username
    "password"              // Password
  ]
}
Server Response 
// Server confirms authorization
{
  "id": 2,
  "result": true,  // Auth successful
  "error": null
}

// Or if authentication fails:
// { "error": ["Invalid user", ...] }
