How V2 Blocks V1 Attacks

set_extranonce Injection

MITM injects malicious pool parameters to redirect hashrate silently.

Encrypted + Authenticated Channel

Noise encryption prevents reading session state. AEAD authentication tags reject any injected frames instantly.

client.reconnect Hijack

Attacker forces miner to connect to a malicious endpoint.

Server Identity Verification

Miners verify the pool's static public key. Unauthorized endpoints cannot complete the Noise handshake.

Share / Job Tampering

Modifying jobs or shares to steal rewards or censor txs.

Digital Signatures

Job Negotiation requires cryptographic signatures. Any modification invalidates the template commitment.

Operational Notes
Key Pinning

Pools distribute their long-term static public key via secure channels (HTTPS/DNSSEC). Miners configure pool_pubkey to reject unknown certs.

Audit Logs

Auth failures generate alerts. Logs capture handshake errors (e.g., Noise_HandshakeFailed), helping detect active MITM attempts.

Fallback Policy

If V2 handshake fails, fallback to V1 only if explicitly allowed by config. Strict mode prevents downgrade attacks.
