SECURITY ANALYSIS // THREAT LANDSCAPE

Protocol Security Issues

Fundamental design choices in Stratum V1 that create significant attack surfaces for miners and pools.

Core Vulnerabilities

  • ▸
    Plaintext by Default

    Communication occurs over raw TCP sockets. No native encryption means all traffic (including mining.authorize credentials) is visible to anyone on the network path.

  • ▸
    No Message Integrity

    Lack of digital signatures or HMACs allows Man-in-the-Middle (MITM) attackers to modify payloads like extranonce1 or difficulty without detection.

  • ▸
    Weak Authentication Binding

    Authorization is a simple message exchange. It does not cryptographically bind the TCP session to the user identity, allowing connection hijacking.

MITM Susceptibility

Because the protocol trusts the transport layer blindly, an on-path attacker can intercept and modify traffic in real-time.

Interception Injection

Sensitive Data at Risk

Values critical to mining operations and revenue are transmitted in cleartext:

extranonce1
difficulty
job_id
worker_pass

Replay & DoS Potential

Without nonces or timestamps in the protocol wrapper, valid messages can be replayed to flood the server or confuse the state machine.
