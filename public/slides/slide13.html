ATTACK VECTOR // METHOD 02

Time-Segment Reconnect

Exploiting connection management to redirect hashrate windows

Concept

Instead of permanently hijacking the miner, the attacker uses the client.reconnect message to periodically force the miner to switch pools.

  • Segmented Mining: The attacker splits mining time between the legitimate pool and their malicious pool.
  • Command Injection: The miner blindly obeys the reconnect command as it looks like a standard load-balancing request.

Stealth Factors

By keeping the "malicious" windows short, the drop in hashrate at the legitimate pool can be mistaken for:
