PROTOCOL MESSAGES // STEP 04

Difficulty & Submission

Dynamic difficulty adjustment and proof of work submission

Set Difficulty

Sent by the server to adjust the share target difficulty.

  • Applies to all subsequent jobs sent to the miner.
  • Allows pool to manage share frequency (keeps connection stable).

Submit Share

Miner sends this when a valid nonce is found (hash < target).

worker_name Previously authorized user.
job_id ID from mining.notify.
nonce The winning nonce value.
Server Notification: Set Difficulty 
// Pool updates difficulty to 2.0
{
  "id": null,
  "method": "mining.set_difficulty",
  "params": [
    2.0  // New Difficulty Value
  ]
}
Client Request: Submit Share 
// Miner submits a valid share
{
  "id": 4,
  "method": "mining.submit",
  "params": [
    "user.worker1",  // Worker Name
    "job123",        // Job ID
    "00000001",      // Extranonce2
    "6502babe",      // nTime
    "e920d3f1"       // Nonce
  ]
}

// Server Response
{ "id": 4, "result": true, "error": null }
