PROTOCOL MESSAGE // STEP 01

mining.subscribe

Initiates connection and retrieves session parameters

Purpose

The first message sent by the miner to the pool. It registers the client for server-push notifications (jobs).

  • Sends User Agent string (software version).
  • Optional: Resume session with previous session ID.

Response Structure

The result array contains three critical components:

Subscriptions

Array of notification types (mining.notify) and their session IDs.

Extranonce1

Hex-encoded unique string for this connection. Part of coinbase serialization.

Xnonce2_Size

Integer size (bytes) of the extranonce2 the miner must generate.

Client Request 
// Miner sends subscription request
{
  "id": 1,
  "method": "mining.subscribe",
  "params": [
    "bfgminer/3.1.0" // User Agent
  ]
}
Server Response 
// Server accepts and assigns IDs
{
  "id": 1,
  "result": [
    [ // Subscriptions
      ["mining.set_difficulty", "sub_id1"],
      ["mining.notify", "sub_id2"]
    ],
    "08000002", // Extranonce1 (Hex)
    4           // Extranonce2_size (Bytes)
  ],
  "error": null
}
