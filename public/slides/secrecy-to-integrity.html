From Secrecy to Integrity: AEAD

The Malleability Problem

Encryption hides data but doesn't protect it. Without integrity checks, an attacker can modify ciphertext in transit (e.g., flipping bits) to alter the decrypted message without being detected.

Authenticated Encryption

Send : ( Enc , Tag )

Tag = Hash( Enc ++ SecretKey )

Verification Process

Upon receipt, the receiver computes their own tag using the same key:
Tag' = H(ReceivedEnc ++ Key).
If Tag' == Tag, the message is authentic.

Why Forgery Fails

A MITM attacker can see Enc and modify it to MaliciousEnc. However, they cannot calculate the correct Tag because they don't know the SecretKey included in the hash.

AEAD in Noise

Noise Protocol uses ChaCha20-Poly1305 (or AES-GCM). This combines encryption and authentication into a single efficient operation, ensuring both confidentiality and integrity.

The Result

Any tampering with the message results in a verification failure. The receiver simply discards the packet, preventing "bit-flipping" attacks or malicious command injection.
