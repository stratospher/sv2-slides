VULNERABILITY ANALYSIS // ROOT CAUSES

Why These Attacks Work

Fundamental design choices in 2012 created the security gaps exploited today.

Plaintext Communication

  • ▸

    No Default TLS: Stratum V1 operates over raw TCP sockets. All data, including usernames and job details, is sent in cleartext.

  • ▸

    Traffic Visibility: Any on-path observer (ISP, rogue router, Wi-Fi admin) can read the JSON-RPC messages and identify mining traffic.

Lack of Message Integrity

  • ▸

    No Authentication Binding: The mining.authorize step authenticates the session but does not cryptographically bind subsequent messages to that session.

  • ▸

    Zero Tamper Evidence: An attacker can modify parameters like extranonce1 or inject client.reconnect commands without the miner detecting any anomaly.

Implicit Trust Model

The protocol assumes the network layer is secure. Clients blindly accept server control messages.

Client: "Server says reconnect? Okay."
Attacker: "I am the server now."

Extranonce Control

The extranonce parameters dictate the valid search space for a block.

  • If you control extranonce...
  • You control the shares.

These are not bugs in implementation, but architectural limitations of a 2012 protocol.
